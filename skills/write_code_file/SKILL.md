## git

- 在开始写代码文件之前，请先检查是否是git仓库
- 如果是，则检查现有的更改是否都已暂存
- 如果未暂存则暂存
- 目的是确保：现在工作区是空的，所有更改都已暂存，不存在未追踪、已追踪但未暂存的更改
- 如果你检查发现：现有的更改都已暂存或者不是git仓库，那么可以继续

## 写代码文件

- 你在写代码文件的时候，如果改动的地方较多、较零碎，则用完整的代码全文覆盖那个文件。
- 如果改动的地方较少、成块、不零碎，则基于需要修改的那个部分的位置（例如行号范围），将需要修改的那个部分替换为修改后的代码。
- 如果涉及大规模重构，在可以使用复制粘贴的情况下使用复制粘贴来移动代码片段，而不是原样将它们输出一遍。
- `references/doc.md`里面是写代码文件的命令以及用法，供你参考。
